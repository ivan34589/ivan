Команды Git
Команда Описание
git init Инициализирует новый репозиторий Git. Используется для инициализации
git init Инициализирует новый репозиторий Git
git clone url Создает копию существующего репозитория Git
git add file Переносит изменения из рабочего каталога в раздел проиндексированных файлов
git add . Переносит изменения из рабочего каталога в раздел проиндексированных файлов
git commit -m "comment" Фиксирует проиндексированные изменения и сохраняет их в историю версий
git commit -a -m "comment" Делает снимок состояния проекта на текущий момент времени.
git push url Команда git push противоположна команде извлечения (с некоторыми оговорками). С ее помощью можно перенести локальную ветку в другой репозиторий и без труда опубликовать поступивший код. Эта команда похожа на svn commit с тем исключением, что она отправляет не один набор изменений, а серию коммитов.
git status Показывает состояние рабочего каталога и проиндексированного снимка состояния
git log Позволяет изучить предыдущие версии проекта. Эта команда имеет несколько параметров форматирования, с помощью которых можно отобразить снимки состояния после выполнения коммитов.
git branch branch_name Эта команда выступает универсальным инструментом администрирования веток. С ее помощью можно создавать изолированные среды разработки в одном репозитории.
git checkout branch_name
git checkout -b branch_name B -git checkout -b позволяет одновременно создать ветку и переключиться на нее. ветка git, когда вы хотите создать ветку, но остаться в текущей ветке. git checkout -b когда вы хотите создать и переключиться.
git remote Удобный инструмент для администрирования удаленных подключений. С его помощью вместо полного URL-адреса в командах fetch, pull и push можно использовать более удобное сокращение.
